# DayDayUp - Spaced Repetition Memory Skill

> 🧠 基于 Obsidian 的智能记忆系统，使用艾宾浩斯遗忘曲线帮助你记住任何知识。

## 配置

在使用前，请设置 vault 路径：

```bash
# 方法 1: 环境变量
export DAYDAYUP_VAULT_PATH="/path/to/your/obsidian/vault/daydayup"

# 方法 2: 运行配置脚本
./setup.sh

# 查看配置状态
./setup.sh --status
```

### 多平台安装说明

**重要**: DayDayUp 采用数据与 Skill 分离的设计，多个 AI 工具共享同一 vault 数据。

- **全局配置**: `~/.daydayup/config.json`（所有平台共享）
- **数据目录**: Obsidian vault（iCloud 同步）

首次安装后，在其他平台安装时：
1. 克隆 skill 到该平台
2. 运行 `./setup.sh --status` 检查配置
3. 无需重新配置，自动使用现有数据和配置

## 触发词

| 触发词 | 行为 |
|--------|------|
| `/daydayup` | 快速呼出 skill，显示帮助和使用选项 |
| `帮我回忆` / `复习` / `今天学什么` | 展示今日待复习内容 |
| `记住 <内容>` / `添加记忆 <内容>` | 创建新记忆卡片 |
| `记忆状态` | 显示学习统计和进度 |

## 核心算法

### 艾宾浩斯间隔计算

基础间隔（天）：
```
Stage 1: 1 天
Stage 2: 2 天
Stage 3: 4 天
Stage 4: 7 天
Stage 5: 15 天
Stage 6: 30 天
Stage 7: 60 天
Stage 8+: 掌握
```

### 难度调整系数

| 反馈 | 调整系数 | 说明 |
|------|----------|------|
| 简单 | × 1.5 | 间隔延长 50% |
| 一般 | × 1.0 | 标准间隔 |
| 困难 | × 0.5 | 间隔缩短 50% |
| 没记住 | 重置到 Stage 1 | 从头开始 |

### 计算公式

```
nextInterval = baseInterval[currentStage] * difficultyMultiplier
nextReview = today + nextInterval
```

## 复习流程

### 1. 开始复习（用户说"帮我回忆"）

1. 读取 `{vaultPath}/cards/` 目录下所有 `.md` 文件
2. 筛选 `nextReview <= today` 的卡片
3. 按 `nextReview` 升序排序（过期越久越优先）
4. 逐张展示卡片

### 2. 展示卡片

```
📋 今日复习 (3张待复习)

## 1/3: [卡片标题]

**问题**: [问题内容]

---
请在心中回忆答案，然后说"看答案"或"记住了"/"没记住"
```

### 3. 用户反馈处理

| 用户说 | 行为 |
|--------|------|
| `简单` / `太简单了` | difficulty=easy, stage+1, 计算下次复习时间 |
| `一般` / `还行` / `记住了` | difficulty=medium, stage+1, 计算下次复习时间 |
| `困难` / `有点难` | difficulty=hard, stage+1 (或不增加), 计算下次复习时间 |
| `没记住` / `忘了` | 重置 stage=1, nextReview=tomorrow |
| `看答案` | 展示答案和 AI 记忆技巧 |

### 4. AI 记忆技巧生成

每次复习后，AI 生成以下记忆辅助（如有需要）：

**记忆技巧：**
- **联想**: 与已知概念的关联
- **类比**: 生活中的类似场景
- **口诀**: 简短易记的助记符
- **例子**: 具体的应用场景
- **可视化**: 心理图像建议

**第一性原理：**
- **本质问题**: 这个知识点解决的根本问题是什么？
- **核心假设**: 这个理论/方法基于哪些最基本的假设？
- **底层逻辑**: 从最基础的原理出发，如何推导出这个结论？
- **边界条件**: 这个知识点在什么条件下成立？什么条件下会失效？

### 5. 记录复习历史

将本次复习记录追加到 `{vaultPath}/reviews/YYYY-MM/` 目录的 session 文件中。

## 添加新记忆

### 用户说"记住 <内容>"

1. 提取内容，生成合适的标题
2. 创建新的 Markdown 文件到 `{vaultPath}/cards/` 目录
3. 设置初始属性：
   - `stage: 0`
   - `nextReview: tomorrow`
   - `difficulty: medium`
4. AI 生成记忆技巧并写入文件
5. 确认创建成功

## 路径解析优先级

1. 环境变量 `DAYDAYUP_VAULT_PATH`
2. 配置文件 `~/.daydayup/config.json`
3. 自动检测常见 Obsidian vault 路径
4. 提示用户输入

## 配置检测流程

在执行任何记忆操作前，AI 必须先检查配置状态：

### 检查优先级

1. 读取 `~/.daydayup/config.json` 中的 `vaultPath`
2. 读取环境变量 `DAYDAYUP_VAULT_PATH`
3. 都没有 → 进入对话式配置流程

### 对话式配置流程

当检测到用户未配置 vault 路径时，按以下流程引导：

1. **询问路径**：
   ```
   🤔 看起来你还没设置记忆存储位置。

   请问你想把记忆卡片保存在哪个目录？
   （例如：/Users/xxx/Documents/obsidian/daydayup）
   ```

2. **用户输入后验证**：
   - 检查路径格式是否有效
   - 如果目录不存在，询问是否创建

3. **创建目录结构**：
   - 创建主目录（如不存在）
   - 创建子目录：cards/, reviews/, templates/

4. **保存配置**：
   创建/更新 `~/.daydayup/config.json`：
   ```json
   {
     "vaultPath": "<用户输入的路径>",
     "configuredAt": "<当前时间 ISO 格式>",
     "version": "1.0.0"
   }
   ```

5. **确认并继续**：
   ```
   ✅ 已设置记忆路径！
   📂 存储位置: <路径>

   <继续执行用户原始请求>
   ```

### 修改记忆路径

当用户说"修改记忆路径"或"设置记忆路径"时：

1. 显示当前配置的路径
2. 询问新路径
3. 更新配置文件
4. 提示用户手动迁移旧数据（如有）

## 安全边界

**重要**: 此 Skill 只能访问和修改配置的 vault 目录：
- `{vaultPath}/cards/`
- `{vaultPath}/reviews/`
- `{vaultPath}/templates/`

**禁止**:
- 读取或修改 vault 路径外的任何文件
- 执行任何系统命令
- 访问网络（除 AI 生成内容外）
